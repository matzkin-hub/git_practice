## 「司令塔発動！イベント委譲魔法を使いこなせ」

## ミッション：後から追加した依頼カードにも、削除ボタンがちゃんと効くようにしよう！

この課題では「動的に増える要素を扱う」「親要素でクリックをまとめて受け取る」技術を習得します。

### コードを写そう

HTMLタブに下のコードを貼り付けてください。

```html
<div class="panel">
  <p class="title">ギルド依頼ボード</p>

  <div class="controls">
    <input id="taskInput" type="text" placeholder="依頼名を入力" />
    <button id="addBtn">追加</button>
  </div>

  <ul id="taskList"></ul>
</div>
```

JSタブに下のコードを貼り付けてください。

```js
const taskInput = document.querySelector('#taskInput');
const addBtn = document.querySelector('#addBtn');
const taskList = document.querySelector('#taskList');

addBtn.addEventListener('click', function() {
  const taskName = taskInput.value.trim();

  if (taskName === '') {
    return;
  }

  const li = document.createElement('li');
  li.innerHTML =
    '<span>' + taskName + '</span>' +
    '<button class="deleteBtn">削除</button>';

  taskList.appendChild(li);
  taskInput.value = '';
});

taskList.addEventListener('click', function(event) {
  const deleteBtn = event.target.closest('.deleteBtn');

  if (!deleteBtn) {
    return;
  }

  const item = deleteBtn.closest('li');
  item.remove();
});
```

追加した依頼カードの削除ボタンが、あとから作ったものにも効くことを確認してください。

### コードを読み解こう

#### 動的に作ったボタンにあとから対応したい

依頼カードは、ボタンを押すたびに新しく作られます。

```js
const li = document.createElement('li');
taskList.appendChild(li);
```

このとき、最初のページ読み込み時点では削除ボタンはまだ存在していません。

#### 親要素にイベントを付ける

```js
taskList.addEventListener('click', function(event) {
```

削除ボタンそれぞれにイベントを付ける代わりに、外側の `taskList` に1つだけイベントを付けています。

こうすると、あとから増えた子要素のクリックもまとめて受け取れます。

#### `closest()` ：近くにある目的の要素を探す

```js
const deleteBtn = event.target.closest('.deleteBtn');
```

クリックした場所がボタンそのものでも、中の文字でも、近くに `.deleteBtn` があればそのボタンを見つけてくれます。

見つからなければ `null` になるので、

```js
if (!deleteBtn) {
  return;
}
```

で「削除ボタン以外は無視する」と書けます。

#### どのカードを消すかも `closest()` で見つける

```js
const item = deleteBtn.closest('li');
item.remove();
```

押された削除ボタンが入っている `<li>` を探し、そのカードだけ消しています。

```js
if (taskName === '') {

}
```

ここでメッセージを出すようにすると、使いやすくなります。
