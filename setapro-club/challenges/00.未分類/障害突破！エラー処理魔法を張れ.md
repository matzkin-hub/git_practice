## 「障害突破！エラー処理魔法を張れ」

## ミッション：入力ミスや取得失敗に備えて、エラーメッセージと再試行を用意しよう！

この課題では「`try / catch` で失敗を受け止める」「`throw` で自分からエラーを投げる」技術を習得します。

この課題では、2026年4月時点で公開されている JSONPlaceholder のサンプルAPIを使います。

### コードを写そう

HTMLタブに下のコードを貼り付けてください。

```html
<div class="panel">
  <p class="title">記事取得ガード</p>

  <input id="postIdInput" type="number" placeholder="1〜100を入力" />
  <button id="loadBtn">取得する</button>
  <button id="retryBtn" style="display: none;">もう一度試す</button>

  <p id="status">記事番号を入れてください</p>
  <h3 id="postTitle">タイトルはここに出ます</h3>
  <p id="postBody">本文はここに出ます</p>
</div>
```

JSタブに下のコードを貼り付けてください。

```js
const postIdInput = document.querySelector('#postIdInput');
const loadBtn = document.querySelector('#loadBtn');
const retryBtn = document.querySelector('#retryBtn');
const status = document.querySelector('#status');
const postTitle = document.querySelector('#postTitle');
const postBody = document.querySelector('#postBody');

async function loadPost() {
  try {
    retryBtn.style.display = 'none';
    status.textContent = '読み込み中です...';
    postTitle.textContent = 'タイトルはここに出ます';
    postBody.textContent = '本文はここに出ます';

    const postId = Number(postIdInput.value);

    if (Number.isNaN(postId) || postId < 1 || postId > 100) {
      throw new Error('記事番号は1〜100で入力してください');
    }

    const response = await fetch('https://jsonplaceholder.typicode.com/posts/' + postId);

    if (!response.ok) {
      throw new Error('通信に失敗しました');
    }

    const post = await response.json();

    if (!post.title) {
      throw new Error('記事が見つかりません');
    }

    postTitle.textContent = post.title;
    postBody.textContent = post.body;
    status.textContent = '記事番号 ' + postId + ' を取得しました';
  } catch (error) {
    status.textContent = error.message;
    retryBtn.style.display = 'block';
  }
}

loadBtn.addEventListener('click', function() {
  loadPost();
});

retryBtn.addEventListener('click', function() {
  loadPost();
});
```

正しい番号なら記事が表示され、範囲外の番号ならエラーメッセージと再試行ボタンが出ることを確認してください。

### コードを読み解こう

#### `try / catch` ：失敗しても止まらないようにする

```js
try {
  // 危ないかもしれない処理
} catch (error) {
  // 失敗したときの処理
}
```

通信や入力値は、いつも期待通りとは限りません。`try / catch` で包むと、失敗したときも画面に説明を出せます。

#### `throw new Error(...)` ：自分からエラーを作る

```js
throw new Error('記事番号は1〜100で入力してください');
```

入力値がおかしいときは、通信に行く前に自分でエラーを投げています。

#### `response.ok` ：通信が成功したかを確認する

```js
if (!response.ok) {
  throw new Error('通信に失敗しました');
}
```

HTTPの結果が失敗なら、ここで止めて `catch` へ渡します。

#### エラー時は次の行動を用意する

```js
retryBtn.style.display = 'block';
```

「失敗しました」だけで終わるより、もう一度試せる方が親切です。
